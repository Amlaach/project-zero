## Web UI

| Principle | Verdict | Notes |
|---|---|---|
| Visual hierarchy | ✅ | Streaming reply is the most prominent element; topbar/model-info are subdued (smaller text, `--text-dim`). |
| Consistent spacing scale | ✅ | 0.4/0.5/0.6/0.75/1/1.25rem reused throughout; no one-off values found in review. |
| Contrast (WCAG AA) | ✅ | Light: `#1a1d23` on `#f7f8fa`; dark: `#e6e8ee` on `#0b0d12` — both comfortably exceed 4.5:1. Dimmed text remains legible in both themes (`01-empty-state`, `05-dark`). |
| Responsive | ⚠️ partial | Verified at 900–1000px desktop widths only; flexbox/relative units should reflow at narrower widths (message bubbles cap at `42rem`, textarea flexes), but no screenshot below 900px was captured — disclosed gap, not blocking. |
| Clear affordances | ✅ | Send/Stop/Params/Theme all have visible borders or fill + hover states; no unlabeled icon-only controls except the (titled, `aria-label`led) image-upload attach icon. |
| Predictable state | ✅ | Empty state, streaming cursor (`02-generating`), distinct Stop button while generating, error state styled distinctly (red, untested visually here but present in code). |
| Anti-"generic AI" checklist | ✅ | No purple/gradient hero, no 3-card layout, no glassmorphism, single accent used purposefully (online dot / active state / primary button only). **Caught and fixed during this review**: `ImageUpload.svelte` used a 📎 emoji as its only icon — replaced with a proper inline SVG (see `docs/ai/change-trace.md`). |
| Symmetry/balance | ✅ | Intentional asymmetry (user right / assistant left) preserved, not "fixed" into false symmetry; topbar left/right groups share a baseline. |
| Gestalt principles | ✅ | Proximity (role label ↔ content), similarity (fill vs. border distinguishes roles), figure-ground (composer separated by border+solid bg), contrast (accent reserved for state, not decoration), Fitts's law (buttons ~40px tall). |

## Bug found and fixed during this review (not a design nit — a real defect)

The live tok/s indicator's `\r`+overwrite was clobbering the actual streamed response text, because both shared the same terminal row with no forced newline between them — every stats update erased the response tokens printed so far. This was only caught by looking at a real screenshot, not by the unit tests (which test formatting logic, not terminal-row interaction). Fixed in `src/cli/live_stats.c` by saving the cursor, jumping to the terminal's last row, printing the stats there, and restoring the cursor — a standard "status line" technique that never touches the response's own cursor position. See `docs/ai/mistakes.md`/`change-trace.md` (2026-07-15) for the full writeup.

## Verdict

All principles pass, with one disclosed, non-blocking gap (narrow-viewport screenshots not
captured). One real bug (live-stats overwrite) and one design-checklist violation (emoji icon)
were found and fixed during this pass — exactly the loop the design-QA gate is meant to catch.
